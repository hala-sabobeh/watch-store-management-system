# Watch Store Management System

## About The Project
This project presents a complete software engineering specification and design for a watch store management system. The system supports both online and in-store operations, including inventory management, order processing, delivery tracking, and after-sales services.

The documentation follows a structured software development process, covering requirements analysis, system modeling, and architectural design.

## System Capabilities
- **Online Shopping**: Browse watches, add to cart, checkout with payment options
- **In-Store Sales**: POS interface for staff-assisted purchases
- **Inventory Control**: Real-time stock tracking and availability updates
- **Delivery Management**: Driver app integration with order tracking
- **After-Sales Support**: Warranty handling, repairs, and returns
- **Customer Reviews**: Verified purchase reviews and ratings

## Documentation Structure

**Requirements Analysis**
- Business description and operational scope
- 8 user requirements with detailed system specifications
- 5 complete scenarios covering normal, alternative, and error flows
- 7 actors identified with their roles and interactions

**System Modeling**
- Use case diagram with 8 functional use cases
- Activity diagrams for core workflows
- Class diagrams (analysis and detailed levels)
- Sequence diagrams for main system interactions

**System Design**
- Component-based architecture with 5 independent services
- Layered design separating presentation, business logic, and data access
- Deployment architecture supporting both web and mobile interfaces
- Design goals: low coupling, high cohesion, and performance optimization
